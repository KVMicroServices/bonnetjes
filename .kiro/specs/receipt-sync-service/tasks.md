# Implementation Plan: Receipt Sync Service

## Overview

Background sync service that polls the KV platform API for reviews, resolves receipt files from S3, and persists them as Receipt records visible in the admin panel. Implemented as a setInterval-based loop within the Next.js process using Prisma for state tracking.

## Tasks

- [x] Add Prisma models (ReceiptSyncState, ReceiptSyncWatermark, ReceiptSyncTick) to `prisma/schema.prisma` with all fields and indexes from the design, run `npx prisma migrate dev` to generate the migration, and add all new environment variables (KV_API_BASE_URL, KV_PUBLICATION_API_TOKENS, KV_RECEIPT_S3_BUCKET_NAME, KV_RECEIPT_AWS_REGION, KV_RECEIPT_AWS_ACCESS_KEY_ID, KV_RECEIPT_AWS_SECRET_ACCESS_KEY, POLL_INTERVAL_SECONDS, WATERMARK_SAFETY_SECONDS, PAGE_SIZE, WORKER_CONCURRENCY, API_RATE_LIMIT, RECEIPT_AUTO_VERIFY_ENABLED, MAX_RETRY_ATTEMPTS) to `.env.example` with comments. _Requirements: 3.1, 3.2, 10.1, 10.2_
