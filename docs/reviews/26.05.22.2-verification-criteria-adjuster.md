## Localization & Stability

### 1. Cached rejected seeding promise (race condition)
**File**: `lib/services/failure-reason-service.ts` ~line 117
**Issue**: If `performSeeding()` rejects, the rejected promise is cached forever. Service becomes permanently broken until restart.
**Fix**: Clear `seedingPromise` in a `.catch()` handler before re-throwing.

### 2. `deleteOverride` throws on non-existent record
**File**: `lib/services/email-template-override-service.ts` ~line 155
**Issue**: Prisma `.delete()` throws `P2025` if no record exists. PATCH handler calls this when clearing fields that were never overridden, causing 500.
**Fix**: Use `deleteMany` (no-op on zero matches) or catch `P2025` and treat as no-op.

## General

### 1. Inconsistent failure reason filtering in streaming/dispute routes
**File**: `app/api/receipts/[id]/ocr/route.ts` line 140, `app/api/dispute/verify/route.ts` line 179
**Issue**: `parseOcrResult` called without `allowedReasons` — disabled reasons still accepted from LLM.

### 2. Static `FAILURE_REASONS` used for secondary analysis validation
