## Security

### 1. Missing input validation on generate endpoint
**File**: `app/api/admin/failure-reasons/generate/route.ts`, lines 36-37
**Issue**: `code` field has no format/length validation before being sent to the AI API.
**Fix**: Apply code format validation (`^[A-Z][A-Z_]*[A-Z]$`, 2-50 chars) or at minimum enforce a max length.

### 2. No length validation on prompt criteria strings
**File**: `app/api/admin/settings/route.ts`, lines 163-174
**Issue**: `ocrPromptCriteria` and `secondaryPromptCriteria` have no max length. Arbitrarily large strings could be stored and sent to the AI API on every request.
**Fix**: Add a max length check (e.g., 50,000 chars) before persisting.

Both are low-severity (admin-only access required).

## Localization & Stability

### 1. Resource Leak — Timer not cleared on fetch failure
**File**: `lib/services/failure-reason-translator.ts`, lines ~97-110 and ~200-213
**Issue**: When `fetch()` throws, `clearTimeout(timeoutId)` is never called. Timer fires `abort()` on a dead controller.
**Fix**: Add `clearTimeout(timeoutId)` at the start of each `catch` block.

### 2. Race Condition — Seeding not guarded against concurrent access
**File**: `lib/services/failure-reason-service.ts`, lines 101-154
**Issue**: Concurrent requests can both enter `ensureBuiltInReasonsSeeded()` before `seedingComplete` is set, risking unique constraint errors.
**Fix**: Add a module-level `Promise` guard so concurrent callers await the same operation.

Translations: All 8 language files updated. No hardcoded user-visible strings found. ✅
