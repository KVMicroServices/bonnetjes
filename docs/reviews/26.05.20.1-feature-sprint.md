## Security

**Issue 1 (High): SMTP password exposed in API response**
- File: `app/api/admin/settings/route.ts`, line 40
- The GET handler returns `getAppSettings()` which includes `smtp.smtpPass` in plaintext JSON. The frontend pre-fills it in a form field.
- Fix: Redact `smtpPass` in the GET response (return `"***"` or `null`). Only accept the value on PATCH.

**Issue 2 (Medium): SMTP password logged in plaintext**
- File: `app/api/admin/settings/route.ts`, line 146
- `logger.info({ settings, ... })` logs the full settings object including `smtp.smtpPass`.
- Fix: Omit or redact `smtp.smtpPass` before logging.

**Issue 3 (Low): Admin page missing client-side auth guard**
- File: `app/admin/page.tsx`, useEffect and render guard
- The `isAdmin` redirect to `/dashboard` was removed. Any authenticated user can visit `/admin` (API calls still 403, but UI renders).
- Fix: Restore the `if (!isAdmin) router.replace("/dashboard")` guard.

No issues found for: SQL injection, XSS, hardcoded secrets, unvalidated redirects, unsafe deserialization, dependency pinning, suspicious packages.
