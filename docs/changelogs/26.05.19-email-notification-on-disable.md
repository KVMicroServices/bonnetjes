# Bonnetjes v26.05.19

Email notification and receipt dispute system for disabled reviews.

**Branch**: feat/email-notification-on-disable → develop
**Date**: 2026-05-19
**Scope**: 39 files changed, 4368 insertions, 64 deletions

---

## Features

- Added branded HTML email notification sent to reviewers when their review is disabled, with tenant-aware Kiyoh/Klantenvertellen branding
- Added receipt dispute flow where reviewers can upload a new receipt via a signed link, get instant OCR + fraud verification, and request human review as fallback
- Added signed dispute tokens (HMAC-SHA256, 30-day expiry) linking email notifications to the dispute page securely
- Added ReceiptDispute database table tracking each dispute back to its originating review

## Architecture

- Added email service layer with SMTP transport, locale-aware translations, and HTML template rendering
- Added dispute token signing and verification module with fallback to NEXTAUTH_SECRET
- Added Kiyoh review client for resolving reviewer email addresses from the platform API
- Added unit tests for email service, dispute tokens, and Kiyoh review client
- Added startup validation for DISPUTE_TOKEN_SECRET and APP_URL in the queue worker
