# Bonnetjes v26.05.20.2

Feature sprint adding receipt comments, notifications, SMTP settings management, analytics improvements, and Google SSO.

**Branch**: feat/feature-sprint → develop
**Date**: 2026-05-20
**Scope**: 60 files changed, 6145 insertions, 626 deletions

---

## Features

- Added receipt comment system with threaded comments, @-mention autocomplete, and mention notifications
- Added in-app notification bell with real-time notification preferences management
- Added admin dispute management API for handling receipt disputes
- Added review-required receipts endpoint for flagging receipts needing manual review
- Added Google SSO sign-up button alongside existing credentials flow
- Added SMTP settings management via admin settings UI (moved from env vars to database)
- Added user search API endpoint for @-mention autocomplete lookups

## Bugs

- Fixed admin auth guards that were incorrectly allowing non-admin users access to admin-only endpoints
- Fixed SMTP password exposure in the settings API response

## QA

- Remediated 30+ banned code patterns (ternaries, magic values, abbreviations, console.error, chained operations)
- Added missing translations across all 8 language files for new features
- Added fraud risk color class helper to eliminate repeated conditional styling logic
- Added relative time translation support to eliminate hardcoded time strings

## Architecture

- Created `lib/services/comment-service.ts` for comment CRUD with validation and mention notifications
- Created `lib/services/notification-service.ts` for in-app notification delivery
- Created `lib/client-logger.ts` for browser-side structured logging (pino is server-only)
- Added Comment model to Prisma schema with indexes and cascade relations
- Added Notification and NotificationPreference models with supporting migrations
- Moved SMTP configuration from environment variables to database-backed app settings
- Added analytics service support for custom date ranges and granularity options
- Added Google OAuth env var validation at startup via vitest config for test environments
- Added comprehensive unit tests for comment-service (22 tests)
