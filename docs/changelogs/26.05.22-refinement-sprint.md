# Bonnetjes v26.05.22

Feature sprint adding file format support, bookmarks, localized emails, and dispute verification improvements.

**Branch**: feat/refinement-sprint → develop
**Date**: 2026-05-22
**Scope**: 66 files changed, 4235 insertions, 162 deletions

---

## Features

- HEIC, DOC, and DOCX file uploads are now supported with automatic conversion to viewable image formats
- Receipts can be bookmarked for quick access, with a dedicated filter toggle on the admin dashboard
- Search bar on admin dashboard filters receipts by review ID or location ID via ReceiptSyncState
- Transactional emails (receipt verified, dispute verified, dispute final rejection) now send in the location's configured locale fetched from the Kiyoh API
- Dispute verification flow sends localized email notifications on approval or final rejection

## Bugs

- Fixed analytics volume chart not counting receipts with `requires_review` status
- Fixed receipt list tests failing due to missing `receiptSyncState.findMany` mock

## QA

- Cleaned up all banned code patterns (ternaries, short-circuit tricks, magic numbers/strings) flagged in code review
- Pinned new dependencies to exact versions (heic-convert, mammoth, @types/heic-convert)
- Added 100-character max-length guard on receipt search parameter to prevent DoS via expensive LIKE queries
- Removed dead parameters from `buildOcrMessages` and merged duplicate email param interfaces

## Architecture

- New `lib/file-conversion.ts` module handles HEIC→JPEG, DOCX→PNG, and DOC→PNG conversions
- New `lib/review-disable/kiyoh-location-client.ts` fetches location locale from Kiyoh API with fallback
- New Bookmark model with unique constraint on (userId, receiptId) and cascade deletes
- Added `previewStoragePath` column to Receipt and `userId` column to Notification for targeted delivery
- New test suites for file conversion and upload service format validation
