# Bonnetjes v26.05.22.1

## Features

- Custom failure reason management with CRUD, ordering, enable/disable, and AI-powered translation to all 8 languages
- Email template editor allowing admins to override transactional email copy per type and locale, with live preview and AI auto-translate
- AI verification prompts (system prompt, secondary analysis prompt) and receipt max age exposed as editable admin settings
- AI-powered generation of failure reason suggestions based on existing reasons and platform context

## QA

- Resolved all banned code patterns flagged in code review (ternaries, short-circuits, double negation, magic strings)
- Fixed `deleteOverride` crash on non-existent record by switching to `deleteMany`
- Fixed `ensureBuiltInReasonsSeeded` caching a rejected promise indefinitely
- Fixed `parseOcrResult` not receiving dynamic `allowedReasonCodes` in dispute and OCR routes

## Architecture

- New `FailureReasonDefinition` and `EmailTemplateOverride` Prisma models with migrations
- New services: `failure-reason-service`, `failure-reason-translator`, `email-template-override-service`, `email-template-translator`
- Extracted OCR constants (prompts, failure reasons, thresholds) into `lib/services/ocr-constants.ts`
- Refactored `ocr-service.ts` to use dynamic failure reasons from DB instead of hardcoded list
- Email translation loader now queries DB overrides before falling back to message files
- Comprehensive test suites for all new services and API routes
