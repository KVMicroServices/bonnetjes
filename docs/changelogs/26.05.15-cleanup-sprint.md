# Kiyoh Mobile v26.05.15

Major cleanup sprint: async queue processing, review disable automation, OCR improvements, i18n, and UI cleanup.

**Branch**: feat/cleanup-sprint → development
**Date**: 2026-05-15
**Scope**: 63 files changed, 5148 insertions, 2920 deletions

---

## Features

- Added BullMQ async receipt processing with Redis backend, replacing synchronous OCR blocking and fire-and-forget patterns
- Added review-disable queue with audit logging that auto-disables Kiyoh reviews after confirmed OCR rejection
- Added context token exchange step to Kiyoh auth flow, fixing 401 errors on review API calls
- Added next-intl i18n infrastructure with language selector and translations for all 8 languages
- Added configurable secondary AI model (gpt-5.4-mini) for independent OCR rejection review
- Added failure reasons, secondary analysis, and English-only enforcement to OCR judging

## Bugs

- Fixed TOTP generation and added token caching for Kiyoh authentication
- Fixed dashboard not waiting for OCR stream completion before refreshing receipt data
- Removed dead Review Queue tab and fixed admin receipts JSON parsing

## QA

- Moved admin tabs from dashboard into the header for cleaner navigation
- Added infinite scroll pagination and configurable poll interval to the dashboard
- Made Kiyoh platform URLs (auth, review API, tenant ID) configurable via environment variables

## Architecture

- Removed Google Drive import feature entirely (API routes, service, component, and tests deleted)
- Removed manual upload UI from dashboard (kept upload endpoint for future dispute system)
- Added Redis 7 container and queue-worker container to docker-compose
- Added `processingStatus` field and `ReviewDisableAudit` table to Prisma schema
- Resolved 15 code standards violations, 4 localization issues, and 3 stability issues from code review
