# Bonnetjes v26.05.20

UI navigation consolidation, secondary analysis improvements, and admin settings overhaul.

**Branch**: feat/UI-cleanup-and-secondary-analysis-improvements → develop
**Date**: 2026-05-19
**Scope**: 55 files changed, 38782 insertions, 7557 deletions

---

## Features

- Added centralized admin settings page with auto-verify, auto-disable, confidence threshold, and location whitelist controls
- Secondary analysis now runs on all uncertain OCR outcomes (not just rejections) and can overwrite verification status and extracted values
- Added OCR confidence column to the admin review queue table
- Added 15-second auto-refresh polling to the admin receipts queue
- Added location whitelist setting to restrict auto-disable to specific location IDs

## Bugs

- Fixed blank page on reload caused by a redundant `mounted` state guard in Providers that raced with session resolution

## Architecture

- Removed the entire automation workflow system (executor, service, API routes, admin page, and tests)
- Removed moderation queue page and review platform pages (consolidated into main admin view)
- Added `AppSetting` database model and `app-settings-service.ts` for key-value admin configuration
- Simplified confidence logic from two thresholds to a single threshold gate
- Added Zod schema validation on secondary analysis LLM output for type safety
- Added Kiyoh API swagger.json documentation to `docs/resources/`
- Added unit and integration tests for settings service, admin settings route, header navigation, and dashboard redirect
