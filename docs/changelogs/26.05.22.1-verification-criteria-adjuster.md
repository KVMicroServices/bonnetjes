## Features

- Custom failure reason management with CRUD, ordering, enable/disable, and AI-powered translation to all 8 supported languages
- Email template editor allowing admins to override email copy per type/locale with live preview and AI auto-translate
- Exposed AI verification prompts (system prompt, secondary analysis prompt) and receipt max age as editable admin settings
- AI-powered generation of failure reason suggestions based on existing reasons and platform context

## Architecture

- New `FailureReasonDefinition` and `EmailTemplateOverride` Prisma models with migrations
- New service layer: `failure-reason-service`, `failure-reason-translator`, `email-template-override-service`, `email-template-translator`
- Extracted OCR constants (prompts, failure reasons, thresholds) into dedicated `ocr-constants.ts` module
- Refactored `ocr-service.ts` to use dynamic failure reasons from DB instead of hardcoded list
- Comprehensive test coverage for all new services and API routes
